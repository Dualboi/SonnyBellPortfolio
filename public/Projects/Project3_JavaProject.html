Java Server chat room Application

This completed university project is a Java-based chat application featuring both client and administrator interfaces, inspired by platforms like Discord. It is designed to support real-time communication across a centralised, multithreaded server environment.

  • Allows users to create usernames, enabling message attribution and identity tracking.
  • Displays messages from other users in real time, facilitating group discussions.
  • Handles multiple users simultaneously using Java multithreading for stable and responsive communication.
  • Makes use of custom made APIs and a REST API for server communication from terminal, web and GUI.

The web administrator interface provides key server-side control features:

  • Start and stop the server via command line.
  • Manually configure the server port as required during initial setup.
  • Set an admin password at server launch to secure privileged controls and restrict unauthorised access.
  • Access and monitor a real-time server activity log.
  • Display currently connected users via a built-in web interface.
  • Send messages to all connected users, allowing for announcements or important updates.
  • Kick or ban users from the server, providing control over user behaviour and maintaining from the admin web page

Client-side functionality and graphical interface:

  • A modern JavaFX GUI client provides a user-friendly experience similar to native applications.
  • Supports launching via GUI (run-gui.bat) or command line for flexibility.
  • Users can connect to the server, chat, and execute commands such as /help or quit.
  • A interactive multiplayer game started by a command in the chat room with score keeping
